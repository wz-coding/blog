# M

#### 以 M 为首的属性

<Mcard>

### [`mm`][zh-link]{ .mdn-link linkparam=length#mm}
CSS 数据类型，表示距离尺寸
</Mcard>

<Mcard>

### [`margin`][zh-link]{ .mdn-link linkparam=margin}
CSS 简写属性，是 `margin-top`，`margin-right`，`margin-bottom`，和 `margin-left` 的简写，为给定元素设置所有四个（上下左右）方向的外边距属性
</Mcard>

<Mcard>

### [`margin-block`][zh-link]{ .mdn-link linkparam=margin-block}
定义了元素的逻辑块首和块末外边距，并根据元素的书写模式、行内方向和文本朝向对应至实体外边距
</Mcard>

<Mcard>

### [`margin-block-end`][zh-link]{ .mdn-link linkparam=margin-block-end}
定义了元素的逻辑块末外边距，并根据元素的书写模式、行内方向和文本朝向对应至实体外边距
</Mcard>

<Mcard>

### [`margin-block-start`][zh-link]{ .mdn-link linkparam=margin-block-start}
定义了元素的逻辑块首外边距，并根据元素的书写模式、行内方向和文本朝向对应至实体外边距
</Mcard>

<Mcard>

### [`margin-bottom`][zh-link]{ .mdn-link linkparam=margin-bottom}
设置元素的底部外边距
</Mcard>

<Mcard>

### [`margin-inline`][zh-link]{ .mdn-link linkparam=margin-inline}
CSS 简写属性，定义了元素的逻辑行首和行末外边距，并根据元素的书写模式、行内方向和文本朝向对应至实体外边距
</Mcard>

<Mcard>

### [`margin-inline-end`][zh-link]{ .mdn-link linkparam=margin-inline-end}
定义元素的逻辑行末外边距，根据元素的书写模式、行内方向和文本朝向映射为实体外边距
</Mcard>

<Mcard>

### [`margin-inline-start`][zh-link]{ .mdn-link linkparam=margin-inline-start}
定义元素的逻辑行首外边距，根据元素的书写模式、行内方向和文本朝向映射为实体的外边距
</Mcard>

<Mcard>

### [`margin-left`][zh-link]{ .mdn-link linkparam=margin-left}
设置与元素相关联的盒子模型的左外边距。这个值可以为负值
</Mcard>

<Mcard>

### [`margin-right`][zh-link]{ .mdn-link linkparam=margin-right}
设置与元素相关联的盒子模型的右外边距。这个值可以为负值
</Mcard>

<Mcard>

### [`margin-top`][zh-link]{ .mdn-link linkparam=margin-top}
设置元素的顶部外边距外边距区域。正值使它离相邻元素更远，而负值使它更靠近相邻元素
</Mcard>

<Mcard>

### [`margin-trim`][en-link]{ .mdn-link linkparam=margin-trim}
允许容器修剪其子项与容器边缘相邻的边距
</Mcard>

<Mcard>

### [`::marker`][zh-link]{ .mdn-link linkparam=::marker}
CSS 伪元素，匹配列表的标记框（通常为一个符号或数字）
</Mcard>

<Mcard>

### [`mask`][zh-link]{ .mdn-link linkparam=mask}
允许使用者通过遮罩或者裁切特定区域的图片的方式来隐藏一个元素的部分或者全部可见区域
</Mcard>

<Mcard>

### [`mask-border`][zh-link]{ .mdn-link linkparam=mask-border}
创建一个紧贴元素边框边缘的 `mask`
</Mcard>

<Mcard>

### [`mask-border-mode`][zh-link]{ .mdn-link linkparam=mask-border-mode}
指定使用的混合模式遮罩边界
</Mcard>

<Mcard>

### [`mask-border-outset`][en-link]{ .mdn-link linkparam=mask-border-outset}
指定元素的遮罩边框距其边框的距离
</Mcard>

<Mcard>

### [`mask-border-repeat`][zh-link]{ .mdn-link linkparam=mask-border-repeat}
置如何调整源图像的边缘区域以适应元素遮罩边框的尺寸
</Mcard>

<Mcard>

### [`mask-border-slice`][en-link]{ .mdn-link linkparam=mask-border-slice}
将 `mask-border-source` 设置的图像划分为区域。 这些区域用于形成元素遮罩边框的组件
</Mcard>

<Mcard>

### [`mask-border-source`][en-link]{ .mdn-link linkparam=mask-border-source}
设置用于创建元素遮罩边框的源图像
</Mcard>

<Mcard>

### [`mask-border-width`][zh-link]{ .mdn-link linkparam=mask-border-width}
设置元素的遮罩边框的宽度
</Mcard>

<Mcard>

### [`mask-clip`][en-link]{ .mdn-link linkparam=mask-clip}
设置受遮罩影响的区域，元素的绘制内容必须限制在这个区域
</Mcard>

<Mcard>

### [`mask-composite`][en-link]{ .mdn-link linkparam=mask-composite}
表示在当前遮罩层及其下方的遮罩层上使用合成操作
</Mcard>

<Mcard>

### [`mask-image`][zh-link]{ .mdn-link linkparam=mask-image}
设置了用作元素蒙版层的图像
</Mcard>

<Mcard>

### [`mask-mode`][zh-link]{ .mdn-link linkparam=mask-mode}
设置由 `mask-image` 指向的遮罩被视为亮度或阿尔法遮罩
</Mcard>

<Mcard>

### [`mask-origin`][en-link]{ .mdn-link linkparam=mask-origin}
设置遮罩层的原点
</Mcard>

<Mcard>

### [`mask-position`][en-link]{ .mdn-link linkparam=mask-position}
为每个定义的遮罩层图像设置初始位置，相对于由 `mask-origin` 设置的遮罩层图像位置
</Mcard>

<Mcard>

### [`mask-repeat`][en-link]{ .mdn-link linkparam=mask-repeat}
设置遮罩层图像的重复方式。 遮罩层图像可以沿水平轴、垂直轴、两个轴重复，或者根本不重复
</Mcard>

<Mcard>

### [`mask-size`][en-link]{ .mdn-link linkparam=mask-size}
指定遮罩层图像的大小。 可以完全或部分限制图像的大小，以保持其固有比例
</Mcard>

<Mcard>

### [`mask-type`][en-link]{ .mdn-link linkparam=mask-type}
设置 SVG`<mask>` 元素是用作亮度还是 alpha 遮罩层。 它适用于 `<mask>` 元素本身
</Mcard>

<Mcard>

### [`masonry-auto-flow`][en-link]{ .mdn-link linkparam=masonry-auto-flow}
设置了在 CSS 网格布局中使用砌体时项目的放置方式
</Mcard>

<Mcard>

### [`math-depth`][en-link]{ .mdn-link linkparam=math-depth}
描述了数学公式的每个元素的深度概念，相对于该公式的顶级容器。 具体来说，这用于确定 `font-size` 属性的计算值，当其指定值为 math 时
</Mcard>

<Mcard>

### [`math-shift`][en-link]{ .mdn-link linkparam=math-shift}
设置 `MathML` 公式中的上标是否应该通过普通或紧凑的移位来提升
</Mcard>

<Mcard>

### [`math-style`][en-link]{ .mdn-link linkparam=math-style}
设置 MathML 方程应该以正常高度还是紧凑高度呈现
</Mcard>

<Mcard>

### [`matrix()`][zh-link]{ .mdn-link linkparam=transform-function/matrix}
CSS 函数，指定了一个由指定的 6 个值组成的 2D 变换矩阵
</Mcard>

<Mcard>

### [`matrix3d()`][zh-link]{ .mdn-link linkparam=transform-function/matrix3d}
CSS 函数，以 4x4 齐次矩阵的形式定义一个 3D 转换。其结果是一个 `<transform-function>` 数据类型
</Mcard>

<Mcard>

### [`max()`][zh-link]{ .mdn-link linkparam=max()}
CSS 函数，从一个逗号分隔的表达式列表中选择最大（正方向）的值作为属性的值
</Mcard>

<Mcard>

### [`max-block-size`][en-link]{ .mdn-link linkparam=max-block-size}
指定元素在与 `writing-mode` 指定的书写方向相反的方向上的最大尺寸
</Mcard>

<Mcard>

### [`max-height`][zh-link]{ .mdn-link linkparam=max-height}
设置元素的最大高度
</Mcard>

<Mcard>

### [`max-inline-size`][zh-link]{ .mdn-link linkparam=max-inline-size}
根据元素的书写模式定义了元素区块的横向或纵向最大尺寸
</Mcard>

<Mcard>

### [`max-width`][zh-link]{ .mdn-link linkparam=max-width}
设置元素的最大宽度值
</Mcard>

<Mcard>

### [`@media`][zh-link]{ .mdn-link linkparam=@media}
CSS at-rule，可用于基于一个或多个媒体查询的结果来应用样式表的一部分
</Mcard>

<Mcard>

### [`min()`][zh-link]{ .mdn-link linkparam=min()}
从逗号分隔符表达式中选择一个最小值作为 CSS 的属性值
</Mcard>

<Mcard>

### [`min-block-size`][zh-link]{ .mdn-link linkparam=min-block-size}
根据元素的书写模式定义了元素区块的横向或纵向最小尺寸
</Mcard>

<Mcard>

### [`min-height`][zh-link]{ .mdn-link linkparam=min-height}
设置元素的最小高度
</Mcard>

<Mcard>

### [`min-inline-size`][zh-link]{ .mdn-link linkparam=min-inline-size}
根据元素的书写模式定义了元素区块的横向或纵向最小尺寸
</Mcard>

<Mcard>

### [`min-width`][zh-link]{ .mdn-link linkparam=min-width}
设置元素的最小宽度
</Mcard>

<Mcard>

### [`minmax()`][zh-link]{ .mdn-link linkparam=minmax()}
定义了一个长宽范围的闭区间，它与CSS 网格布局一起使用
</Mcard>

<Mcard>

### [`mix-blend-mode`][zh-link]{ .mdn-link linkparam=mix-blend-mode}
设置元素的内容应该与元素的直系父元素的内容和元素的背景如何混合
</Mcard>

<Mcard>

### [`mod()`][en-link]{ .mdn-link linkparam=mod()}
CSS 函数，返回第一个参数除以第二个参数时剩余的模数，类似于 JavaScript 余数运算符 `%`
</Mcard>

<Mcard>

### [`ms`][zh-link]{ .mdn-link linkparam=time#ms}
CSS 数据类型 ，表示以秒（s）或毫秒（ms）为单位的时间的值
</Mcard>
